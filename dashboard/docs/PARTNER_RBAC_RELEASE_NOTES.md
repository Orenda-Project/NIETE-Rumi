## Release Notes Created

### 1. Main Feature (Highlighted)

**Title**: Partner access is now organized and secure!

**Description**:
You can now invite partner organizations to access Rumi's dashboard with precise control over what they see. Partners only view data for their schools, countries, or specific users.

**Details**:
Previously, granting dashboard access meant showing everything to everyone. Now you can invite partner admins and viewers, scope their access to specific schools or countries, and keep sensitive data private. Perfect for regional partners, school networks, and implementation teams who need focused insights.

### 2. Invitation System

**Title**: Email invitations make onboarding simple!

**Description**:
Send branded email invitations to partner admins and viewers. They receive a secure setup link, create their password, and get instant access to their scoped dashboard.

**Details**:
The invitation system handles everything: token generation, expiration (7 days), branded email templates with role badges, and automatic account creation. Partners can't access data outside their scope from day one.

**Category**: feature
**Icon**: mail
**Highlighted**: No

---

### 3. Access Scopes

**Title**: Five ways to scope partner access!

**Description**:
Choose exactly what each partner can see: by country (Pakistan, Sri Lanka), by school name, by specific phone numbers, or combine multiple filters. Full flexibility for your partnerships.

**Details**:
Access scopes are enforced at the database level with Row-Level Security (RLS), so partners physically cannot query out-of-scope data. Super admins can edit scopes anytime with live preview of affected user counts.

**Category**: feature
**Icon**: filter
**Highlighted**: No

---

### 4. Security & Audit

**Title**: Security and compliance built-in!

**Description**:
Every admin action is logged, Row-Level Security prevents data leaks, and self-deactivation is blocked. Partners can only see their scoped data, super admins see everything.

**Details**:
RLS policies enforce access control at the PostgreSQL level—no code bugs can bypass this. All user management actions (invitations, scope changes, deactivations) are logged to the audit_logs table with timestamps and actor details.

**Category**: improvement
**Icon**: shield
**Highlighted**: No
